# ระบบแจ้งลางาน (ฟรี 100%)

เว็บแอปแจ้งลางานสำหรับองค์กรเล็ก-กลาง ใช้งานผ่านมือถือเป็นหลัก (รองรับ PC)

| ส่วน | เทคโนโลยี | ค่าใช้จ่าย |
|---|---|---|
| หน้าเว็บ | GitHub Pages (HTML/CSS/JS ล้วน ไม่มี framework) | ฟรี |
| API | Google Apps Script Web App | ฟรี |
| ฐานข้อมูล | Google Sheets | ฟรี |
| ไฟล์แนบ (ใบรับรองแพทย์) | Google Drive | ฟรี |
| แจ้งเตือน | อีเมล (Gmail ~100 ฉบับ/วัน) | ฟรี |

## ฟีเจอร์

- ยื่นใบลาผ่านมือถือ เห็นวันลาคงเหลือทันที
- **นับวันลาอัตโนมัติ** — ค่าเริ่มต้นนับทุกวัน (บริษัททำงานทั้งสัปดาห์) รองรับลาครึ่งวัน
  ถ้าบริษัทหยุดเสาร์-อาทิตย์ ให้เปลี่ยน `WORK_ALL_WEEK = false` ใน `Code.gs` ระบบจะข้ามเสาร์-อาทิตย์และวันหยุดในแท็บ `Holidays` ให้อัตโนมัติ
- **แนบใบรับรองแพทย์** — อัปโหลดรูป/PDF เก็บใน Google Drive อัตโนมัติ
- หัวหน้าอนุมัติได้ 2 ทาง: กดลิงก์ในอีเมลแล้วยืนยัน (ไม่ต้อง login) หรือแท็บ "อนุมัติ" ในเว็บ
- **Dashboard สำหรับ HR/admin** — สถิติรวม, ปฏิทินดูว่าใครลาวันไหน, ตารางวันลาคงเหลือรายคน
- **เพิ่มพนักงานใหม่จากหน้าเว็บ** — admin กรอกฟอร์มในแท็บ "สรุป" ได้เลย ไม่ต้องเข้า Google Sheets
- ติดตั้งเป็นแอปบนมือถือได้ (PWA — "เพิ่มลงหน้าจอโฮม")
- แจ้งเตือนอีเมลอัตโนมัติ ทั้งตอนยื่นและตอนรู้ผล

## ขั้นตอนติดตั้ง (ประมาณ 15 นาที)

### 1. สร้าง Google Sheet + วางโค้ด backend

1. สร้าง Google Sheet ใหม่ที่ [sheets.new](https://sheets.new) ตั้งชื่อเช่น "ระบบลางาน"
2. เมนู **Extensions > Apps Script**
3. ลบโค้ดเดิม แล้ววางเนื้อหาจากไฟล์ [`apps-script/Code.gs`](apps-script/Code.gs) ทั้งหมด กด Save
4. เลือกฟังก์ชัน **`setup`** จาก dropdown ด้านบน แล้วกด **Run**
   (ครั้งแรกจะขอสิทธิ์ — กด Review permissions > เลือกบัญชี > Advanced > Go to ... (unsafe) > Allow)
   สคริปต์จะสร้างแท็บทั้งหมดพร้อมข้อมูลตัวอย่าง และสร้างโฟลเดอร์ Drive สำหรับไฟล์แนบให้อัตโนมัติ

### 2. Deploy เป็น Web App

1. กดปุ่ม **Deploy > New deployment**
2. กดไอคอนเฟือง เลือกประเภท **Web app**
3. ตั้งค่า:
   - Execute as: **Me** (บัญชีของคุณ)
   - Who has access: **Anyone**
4. กด **Deploy** แล้วคัดลอก **Web app URL** (ลงท้ายด้วย `/exec`)

> ⚠️ "Anyone" หมายถึงใครที่มี URL เรียก API ได้ แต่ทุกคำสั่งต้องผ่าน login (PIN) ก่อนเสมอ ตัว Google Sheet ยังเป็น private ของคุณคนเดียว

### 3. ตั้งค่าหน้าเว็บ + ขึ้น GitHub Pages

1. แก้ไฟล์ [`config.js`](config.js) — วาง Web app URL ที่คัดลอกมา:
   ```js
   const API_URL = 'https://script.google.com/macros/s/AKfycb....../exec';
   ```
2. สร้าง repository ใหม่บน GitHub (public) แล้ว push โค้ดทั้งโฟลเดอร์นี้ขึ้นไป:
   ```
   git init
   git add .
   git commit -m "leave system"
   git remote add origin https://github.com/<user>/<repo>.git
   git push -u origin main
   ```
3. ในหน้า repo: **Settings > Pages > Source: Deploy from a branch** เลือก branch `main` / โฟลเดอร์ `/ (root)` กด Save
4. รอ 1-2 นาที เว็บจะอยู่ที่ `https://<user>.github.io/<repo>/`

### 4. ใส่ข้อมูลจริง

วิธีง่ายสุด: login ด้วยบัญชี admin แล้วใช้ฟอร์ม **"เพิ่มพนักงานใหม่"** ในแท็บ "สรุป" ของเว็บได้เลย (ตั้ง PIN ไปพร้อมกัน)

หรือจะแก้ใน Google Sheet ตรง ๆ ก็ได้:

- **แท็บ `Employees`** — ใส่พนักงานจริง คอลัมน์สำคัญ:
  - `role`: `employee` (พนักงาน) / `approver` (หัวหน้า เห็นแท็บอนุมัติ) / `admin` (HR เห็นทุกอย่าง + dashboard)
  - `manager_id`: รหัสของหัวหน้าที่เป็นคนอนุมัติใบลาของคนนั้น
  - `pin_hash`: **ห้ามพิมพ์ PIN ตรง ๆ** — ไปที่ Apps Script เปิดฟังก์ชัน `setPin` แก้ค่า `EMP_ID` และ `NEW_PIN` แล้วกด Run ทีละคน
- **แท็บ `Holidays`** — เติมวันหยุดบริษัท/วันหยุดชดเชย/วันพระใหญ่ตามประกาศแต่ละปี (รูปแบบ `YYYY-MM-DD`)
- **แท็บ `LeaveTypes`** — ปรับโควตาวันลาตามระเบียบบริษัท (เว้นว่าง = ไม่จำกัด)

### บัญชีตัวอย่างหลัง setup (PIN ทั้งหมดคือ `1234`)

| รหัส | บทบาท |
|---|---|
| EMP001 | พนักงาน |
| EMP010 | หัวหน้า (อนุมัติ) |
| EMP999 | HR/admin (เห็น dashboard) |

> ใช้ทดสอบระบบก่อน แล้วลบทิ้ง/แก้เป็นข้อมูลจริง

## เมื่อแก้โค้ด Apps Script ภายหลัง

ต้องกด **Deploy > Manage deployments > (ไอคอนดินสอ) > Version: New version > Deploy** ทุกครั้ง ไม่เช่นนั้น URL เดิมจะยังรันโค้ดเวอร์ชันเก่า

## โครงสร้างข้อมูลใน Google Sheets

| แท็บ | เก็บอะไร |
|---|---|
| `Employees` | พนักงาน, อีเมล, PIN (hash), บทบาท, หัวหน้า |
| `LeaveRequests` | ใบลาทุกใบ (1 แถว = 1 ใบ) พร้อมสถานะและลิงก์ไฟล์แนบ |
| `LeaveTypes` | ประเภทลา + โควตาต่อปี |
| `Holidays` | วันหยุดบริษัท ใช้ตอนนับวันลา |

วันลาคงเหลือไม่มีแท็บแยก — ระบบคำนวณสดจากโควตาลบใบลาที่อนุมัติ/รออนุมัติของปีนั้น

## ข้อจำกัดที่ควรรู้

- อีเมลแจ้งเตือนส่งได้ ~100 ฉบับ/วัน (โควตา Gmail ฟรี) — พอสำหรับทีมหลักสิบถึงร้อยคน
- ไฟล์แนบถูกตั้งเป็น "anyone with link can view" เพื่อให้หัวหน้าเปิดจากอีเมลได้ — อย่าแนบเอกสารลับ
- ความปลอดภัยระดับพื้นฐาน (PIN + token) เหมาะกับระบบภายใน ไม่เหมาะเก็บข้อมูลอ่อนไหวเช่นเงินเดือน
- โค้ดหน้าเว็บบน GitHub Pages เป็น public — อย่า hardcode ความลับใด ๆ ในโค้ดฝั่งเว็บ (URL ของ API ไม่ใช่ความลับ)
